# NUM.lsp — יומן פיתוח

## מטרת הפרויקט
תוסף
AutoLISP
ל-
ZWCAD
למספור אוטומטי של חדרים / אזורים בשרטוט הנדסי.
הפקודה
`NUM`
ממספרת פוליגונים סגורים ועיגולים, שומרת הגדרות פר קובץ, ומנהלת את הטקסטים לאורך חיי השרטוט.

---

## שלבי הפיתוח

### שלב 1 — ארכיטקטורה בסיסית
- פקודה אחת `NUM` — בפעם הראשונה פותחת הגדרות, אחר כך תפריט
- הגדרות (שכבה, גובה טקסט, פונט, מרחק, קידומת, סיומת, מספר התחלה) נשמרות במילון
XRECORD
בתוך קובץ ה-
DWG
- תפריט:
`[Settings / Objects / Window / Check]`
- בחירה ידנית של אובייקטים (
`Objects`
)
- מיון מימין לשמאל, מלמעלה למטה

### שלב 2 — XData ותיוג
- כל טקסט מקבל
XData
עם: גרסה + מספר +
`handle`
של האובייקט המקורי
- כל אובייקט מממוספר מקבל
XData
עם: גרסה + מספר +
`handle`
של הטקסט
- כך יש קשר דו-כיווני בין טקסט לאובייקט
- עותק של אובייקט ממוספר יורש את אותו המספר ומתעדכן איתו

### שלב 3 — עדכון אוטומטי
- כשמשנים הגדרות (
`Settings`
) — כל הטקסטים הקיימים מתעדכנים מיד:
  שכבה, גובה, סגנון, מרחק, קידומת, סיומת
- כולל עותקים של הטקסטים

### שלב 4 — Window (בחירה במרובע)
- משמאל לימין = חלון (רק אובייקטים שלמים בתוך המרובע)
- מימין לשמאל = חוצה (כל מה שנוגע במרובע)

### שלב 5 — Check (סריקה)
- סורקת כל אובייקט עם
XData
— בודקת אם הטקסט שלו עדיין קיים
- מציגה סיכום: כמה אובייקטים, כמה טקסטים קיימים, כמה חסרים
- מציעה **שחזור** של מספרים שנמחקו
- בודקת **פערים ברצף** מהמספר המוגדר ועד המקסימום
- מציעה **מילוי פערים** — בחירת אובייקט לכל מספר חסר
- מציעה **איפוס מלא** — מחיקת כל המספרים ותחילה מחדש

### שלב 6 — ניקוי וכתיבה מחדש
כתיבה מחדש של הקובץ כולו לאחר צבירת תיקונים רבים, מסיבות:
- חוסר איזון בסוגריים שנצבר מתיקונים רבים
- פקודת
`(exit)`
שגרמה לשגיאות
- כפילויות קוד בין
`Objects`
ל-
`Window`

## בעיות שנפתרו

| בעיה | פתרון |
|---|---|
| `(exit)` גרם לשגיאה `"Error: quit/exit abort"` | הוחלף בדגל `proceed` + `if/progn` תקינים |
| טקסט ישן לא נמחק מהמסך בממספור מחדש | `num:delete-texts-for` — מחפש לפי `orig-handle`, מוחק גם עותקים |
| כפילויות XData | `vl-remove-if '(lambda (x) (= (car x) -3)) ed)` לפני כל כתיבה |
| עיגולים לא עובדים | `num:is-valid-obj` — תמיכה ב-`CIRCLE` בנוסף ל-`LWPOLYLINE` סגור |
| המספור המשיך מהנקודה שנעצר גם אחרי מחיקה | `num:clear-all` מאפס את `last_num` ל-`"0"` |
| Check לא זיהה מספרים חסרים מתחת למינימום | החל מ-`atoi(nth 6 s)` (מספר ההתחלה המוגדר) ולא מהמינימום שנמצא |
| Check הציג "הכל קיים" גם כשהיו חסרים | הודעת "הכל תקין" מוצגת רק כשגם הטקסטים וגם הרצף תקינים |
| מספור בבחירה מרובה — `No` עצר את כל הפקודה | דגל `proceed` — `nil` מדלג על האובייקט, הלולאה ממשיכה |
| קוד כפול בין `Objects` ל-`Window` | הוצא ל-`num:number-list` — פונקציה משותפת |

## כללי התנהגות

### ממספור אובייקט שכבר ממוספר
- הטקסט **קיים על המסך** → שואל "למספר מחדש? [Yes/No]" — גם בבחירה יחידה וגם בבחירה מרובה (לכל אובייקט בנפרד)
- הטקסט **נמחק** → ממשיך ישירות ללא שאלה

### מיון
סבילות שורה: `max(5.0, גובה-טקסט × 2)`
מימין לשמאל, מלמעלה למטה.

## מבנה XData

### על ישות הטקסט (`NUM_TAG`):
```
(1000 . "1")           ; גרסה
(1000 . "5")           ; המספר
(1000 . "A3F")         ; handle של האובייקט המקורי
```

### על האובייקט המקורי (`NUM_TAG`):
```
(1000 . "1")           ; גרסה
(1000 . "5")           ; המספר
(1000 . "B12")         ; handle של הטקסט
```
